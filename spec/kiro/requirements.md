# Requirements (traceable to 商业计划书)

Sources: `三亚旅居民宿平台商业计划书_完整版.md` — 执行摘要、第八章产品与服务体系（旅居时长/社区）、第十一章技术方案、MVP 里程碑（约 3844、3412–3447 行一带）。

## R-1 市场与数据陈述（文档层）

- **R-1.1** 首页、关于页、帮助页中展示的宏观数字须与计划书一致并标注来源（如：三亚市统计局、新华社、海南出入境管理局等）。
- **R-1.2** 不在 UI 中编造未在计划书出现的精确统计；若展示预测须标明「预测/模型假设」。

## R-2 旅居全时长（P0）

- **R-2.1** 搜索与房源卡片支持短租/月租/季租维度（计划书 3414、57 行）。
- **R-2.2** 价格展示同时暴露日价与长住参考价（月/季）之一览或切换。

## R-3 交易闭环（P0）

- **R-3.1** 预订流：日期、人数、租期类型、费用明细（计划书 3417–3418）。
- **R-3.2** 支付：Stripe 意图创建与多币种字段预留（计划书 81、3417）；无密钥时构建不得失败。
- **R-3.3** 资金托管：产品文案与订单状态体现「确认后结算」逻辑（计划书 76、3417）。

## R-4 信任与认证（P0→P1）

- **R-4.1** 房源认证等级（钻石/金牌/银牌/标准）在列表与详情可见（计划书 3414、认证体系）。
- **R-4.2** 适老化与电梯等筛选与标签（计划书 1790–1819 需求矩阵、第五章）。

## R-5 社区（MVP 基础版）

- **R-5.1** 帖子列表、分类、活动入口（计划书 3420、社区化运营 78 行）。
- **R-5.2** 评论与互动 UI；后端以 Supabase 表为准（`community_posts` 等）。

## R-6 国际化（P1）

- **R-6.1** 五语文案文件已存在；逐步替换硬编码中文为 `next-intl`（计划书 81 行）。

## R-7 技术约束（当前实现）

- **R-7.1** Web 主栈：Next.js App Router + Tailwind + shadcn；后端 BaaS：Supabase（PostgreSQL + RLS）；部署：Netlify（计划书 Web 与云原生方向 5335–5341、5280 行 — 本仓库采用 Netlify 而非自建 K8s，为 MVP 刻意简化）。
- **R-7.2** 合规方向：隐私与实名在帮助中心披露；全量合规对接列为 V1+（计划书 5250 行运营合规列表）。

## R-8 AI（P1）

- **R-8.1** 客服 Bot API：`/api/ai/chat`；无 `OPENAI_API_KEY` 时返回 503 与说明（避免生产静默失败）。
